# 回忆书（Memory Book）工具说明 —— 微调数据参考

来源代码：
- 工具定义：`spring_note/rust/src/ai_openai.rs` `memory_tools_json()`
- 系统提示词：`spring_note/rust/src/ai.rs` `MEMORY_TOOL_SYSTEM_PROMPT`
- 工具执行（出参生成）：`spring_note/lib/core/services/memory_search_service.dart` `executeTool()`
- 调用循环：`spring_note/lib/features/memory/memory_page.dart` `_runToolCallingLoop()`

## 1. 系统提示词（原文）

```
你是 SpringNote 的回忆书问答助手。你必须基于用户的历史日报、周报、月报回答问题。
你可以自主调用工具检索或读取记录；需要信息时先调用工具，不要让应用预先替你检索。
连续追问时结合完整消息历史理解省略指代，例如“什么时候”“这个配置”“刚才说的”等。
回答必须只依据工具返回和对话上下文；材料不足时明确说明缺少依据，不要编造事实。
最终回答使用自然中文和清晰 Markdown，不要输出工具调用 JSON。
```

## 2. 调用协议（OpenAI chat/completions 格式）

- 首轮消息：`[{"role":"system","content":<系统提示词>}, ...历史消息, {"role":"user","content":<问题>}]`，请求体带 `tools`（见第 3 节）与 `temperature: 0.2`。
- 模型要调工具时返回：`{"role":"assistant","content":null或文本,"tool_calls":[{"id":"call_xxx","type":"function","function":{"name":<工具名>,"arguments":<JSON字符串>}}]}`（可能附带 `reasoning_content`）。
- 应用在本地执行工具，把结果作为 tool 消息追加：`{"role":"tool","tool_call_id":"call_xxx","content":<JSON字符串>}`，然后再次请求模型。
- 循环直到模型不再返回 tool_calls（输出最终中文 Markdown 回答），或达到轮次上限。
- 轮次上限 = 配置 `memorySearchLimit`，默认 **12**（范围 1–12），达到上限后应用提示用户缩小问题。
- 同一轮会话内相同「工具名+参数」的重复调用会被缓存拦截，tool 消息内容变为：`{"cached":true,"note":"This exact tool call already returned earlier in this conversation. Use the cached result and continue to answer instead of calling the same tool again.","result":<首次返回原文>}`。

## 3. 工具清单（共 9 个，`strict: true`，`additionalProperties: false`）

### 3.1 get_current_date —— 获取当前日期

- description（原文）：`Get the current local date, ISO week label, and week number. Use this before resolving relative dates such as today, yesterday, this week, this month.`
- 入参：无（`properties: {}`，`required: []`）
- 出参：

```json
{"date": "2026-07-19", "isoWeek": "2026-W29", "weekNumber": 29}
```

### 3.2 keyword_search —— 全局关键词搜索

- description：`Run one global indexed search across SpringNote daily, weekly, and monthly Markdown records. Use this only when the record type is unknown or the answer may span multiple types; prefer a scoped search when the user names daily, weekly, or monthly records. When calling keyword_search, submit all distinctive keywords in this single call. Every keyword must contain at least two Unicode characters.`
- 入参：

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `keywords` | array&lt;string&gt; | 必填，1–8 个；每个关键词 `minLength: 2`（至少两个 Unicode 字符） | 一次调用提交全部有区分度的关键词，按重要性排序 |

- 出参：`{"results":[<MemorySource>...]}`（见第 4 节），按匹配次数打分排序，最多返回 `memoryKeywordSearchResultLimit` 条（默认 **12**）。

### 3.3 search_daily_notes —— 只搜日报

- description：`Search only SpringNote daily Markdown notes. Use this instead of keyword_search when the request is limited to daily notes or day-level records. Submit all distinctive keywords in this single call. Every keyword must contain at least two Unicode characters.`
- 入参：同 `keyword_search`（`keywords`，1–8 个，每个 ≥2 字符）
- 出参：同 `keyword_search`，仅含日报。

### 3.4 search_weekly_notes —— 只搜周报

- description：`Search only SpringNote weekly Markdown reports. Use this instead of keyword_search when the request is limited to weekly reports. Submit all distinctive keywords in this single call. Every keyword must contain at least two Unicode characters.`
- 入参/出参：同上，仅含周报。

### 3.5 search_monthly_notes —— 只搜月报

- description：`Search only SpringNote monthly Markdown reports. Use this instead of keyword_search when the request is limited to monthly reports. Submit all distinctive keywords in this single call. Every keyword must contain at least two Unicode characters.`
- 入参/出参：同上，仅含月报。

### 3.6 read_daily_note —— 读某天日报全文

- description：`Read the full daily Markdown note for a specific date.`
- 入参：

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `date` | string | 必填，pattern `^20\d{2}-(0[1-9]|1[0-2])-([0-2][0-9]|3[0-1])$` | 日期，`YYYY-MM-DD` |

- 出参：`{"results":[<MemorySource>]}`（0 或 1 条；title 形如 `日报 2026-07-19`，snippet 为日报全文截断）。

### 3.7 read_week_daily_notes —— 读日期范围内全部日报

- description：`Read all available daily notes in a date range, typically one week.`
- 入参：

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `startDate` | string | 必填，日期 pattern 同上 | 范围起始，`YYYY-MM-DD` |
| `endDate` | string | 必填，日期 pattern 同上 | 范围结束，`YYYY-MM-DD` |

- 出参：`{"results":[<MemorySource>...]}`，范围内存在的日报逐日一条，最多 `memoryWeekDailyNoteLimit` 条（默认 **31**，范围 1–31）。

### 3.8 read_weekly_note —— 读某 ISO 周周报全文

- description：`Read only the full SpringNote weekly report Markdown for a specific ISO week. Do not return daily notes.`
- 入参：

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `week` | string | 必填，pattern `^20\d{2}-W(0[1-9]|[1-4]\d|5[0-3])$` | ISO 周，`YYYY-Www`，如 `2026-W28` |

- 出参：`{"results":[<MemorySource>]}`（0 或 1 条；title 形如 `周报 2026-W28`）。

### 3.9 read_month_weekly_notes —— 读某月覆盖的全部周报

- description：`Read all available SpringNote weekly report Markdown files whose ISO weeks overlap a specific calendar month. Return weekly reports only, not daily notes or the monthly report.`
- 入参：

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `month` | string | 必填，pattern `^20\d{2}-(0[1-9]|1[0-2])$` | 自然月，`YYYY-MM` |

- 出参：`{"results":[<MemorySource>...]}`，与该月有重叠的所有 ISO 周周报（0–6 条左右）。

### 3.10 read_month_report —— 读某月月报全文

- description：`Read only the monthly report Markdown for a specific month. Do not return daily notes.`
- 入参：同 3.9（`month`，`YYYY-MM`）
- 出参：`{"results":[<MemorySource>]}`（0 或 1 条；title 形如 `月报 2026-07`）。

## 4. 出参（tool 消息 content）统一结构

除 `get_current_date` 外，所有工具的 content 都是如下 JSON 字符串：

```json
{
  "results": [
    {
      "title": "日报 2026-07-19",
      "path": "D:/.../daily/2026-07-19.md",
      "snippet": "（命中的 Markdown 内容片段）",
      "score": 100
    }
  ]
}
```

- `title`：搜索类为文件名（去掉 `.md`）；读取类为 `日报/周报/月报 + 日期标签`。
- `path`：本地文件绝对路径。
- `snippet`：
  - 搜索类：去掉空行后的正文，从第一个关键词命中处取窗口（前 `memoryKeywordContextBefore` 默认 **1400** 字符、后 `memoryKeywordContextAfter` 默认 **2600** 字符），最长 `memoryResultMaxCharacters`（默认 **3600**）字符，超出补 `...`。
  - 读取类：全文（去空行），同样截断到 `memoryResultMaxCharacters`。
- `score`：搜索类为关键词命中总次数；读取类为固定值（日报 100、周报 120、月报 140）。
- 无结果：`{"results":[]}`。
- 本地执行异常：`{"results":[],"error":"local_tool_execution_failed"}`。

## 5. 微调数据构造要点

1. 每条样本的 `system` 用第 1 节原文，`tools` 用第 3 节 9 个工具的 JSON Schema（与线上一致，`strict: true`）。
2. 多轮工具调用样本结构：`user → assistant(tool_calls) → tool(JSON) → [可重复] → assistant(最终回答)`。
3. 最终回答约束：自然中文、Markdown、只依据 tool 返回内容；材料不足要明说，不编造；不输出工具调用 JSON。
4. 典型调用模式（可作为正样本路径）：
   - 含相对时间（今天/本周/上月）→ 先 `get_current_date` 再按解析结果调读取类工具。
   - 指明记录类型（日报/周报/月报）→ 用对应 `search_*_notes`，而不是 `keyword_search`。
   - 类型不明或跨类型 → `keyword_search`，命中后可用 `read_daily_note` 等取全文确认。
   - 所有关键词在一次调用中提交（1–8 个、每个 ≥2 字符），不要逐词多次调用。
   - 同一「工具+参数」不重复调用（线上会被缓存拦截）。
